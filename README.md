# Volt Logistics - Telegram Bot + Web App

Система для обработки заказов через Telegram бота и веб-приложение.

## Архитектура проекта

- **Web App** - размещается на GitHub Pages
- **Telegram Bot** - работает на сервере (для тестов на локальном ПК)
- **База данных** - SQLite файл `volt_logistics.db`
- **Заказы** - сохраняются в Excel файлы по дням в папке `/orders/daily/`

## Структура файлов

```
├── bot.py                 # Telegram бот
├── volt_logistics.db      # База данных SQLite
├── orders/                # Папка с заказами
│   ├── daily/            # Ежедневные Excel файлы
│   └── volt_order_table.xlsx  # Пример структуры
├── logs/                  # Логи работы бота
├── index.html            # Главная страница Web App
├── order.html            # Страница создания заказа
├── profile.html          # Страница профиля
├── main.html             # Страница main
├── style.css             # Общие стили
└── script.js             # JavaScript логика
```

## Как работает система

1. **Пользователь** открывает Web App через Telegram бота
2. **Заполняет форму** заказа на странице order.html
3. **Данные отправляются** через Telegram WebApp API в бота
4. **Бот обрабатывает** заказ и сохраняет в:
   - SQLite базу данных
   - Excel файл в `/orders/daily/YYYY-MM-DD.xlsx`
5. **Менеджеры получают** уведомление о новом заказе

## Настройка и запуск

### 1. Настройка бота

1. Установите зависимости:
```bash
pip install telebot sqlite3 pandas openpyxl
```

2. Обновите токен бота в `bot.py`:
```python
BOT_TOKEN = 'ваш_токен_бота'
```

3. Обновите ID менеджеров:
```python
MANAGER_IDS = [ваш_id, id_менеджера2]
```

4. Запустите бота:
```bash
python bot.py
```

### 2. Настройка Web App

1. Загрузите все HTML/CSS/JS файлы на GitHub Pages
2. Обновите URL в боте:
```python
webapp_url = "https://ваш_username.github.io/ваш_репозиторий/"
```

### 3. Настройка WebApp в BotFather

1. Откройте @BotFather в Telegram
2. Выберите вашего бота
3. Нажмите "Bot Settings" → "Menu Button"
4. Установите URL: `https://ваш_username.github.io/ваш_репозиторий/`

## Структура Excel файлов

Каждый день создается новый Excel файл с заказами:
- Путь: `/orders/daily/YYYY-MM-DD.xlsx`
- Колонки: Заказ ID, Фото, Ссылка, Площадка, Цвет, Размер, Цена, Количество, и т.д.

## Функции Web App

- **Главная страница** (index.html) - навигация по приложению
- **Создание заказа** (order.html) - форма для оформления заказа
- **Профиль** (profile.html) - управление профилем пользователя
- **Main** (main.html) - информация о компании и подписка на канал

## Навигация

- Все страницы связаны через функцию `navigateTo()`
- Нижнее меню: Профиль, Чат с менеджером, Создать заказ
- Кнопка "Назад" на всех страницах кроме главной

## Тестирование

1. Запустите бота локально
2. Откройте Web App через бота
3. Создайте тестовый заказ
4. Проверьте:
   - Уведомление менеджерам
   - Запись в базу данных
   - Создание Excel файла в `/orders/daily/`
